📡 ESP32 WiFi Sniffer

Setup Guide — Fontys ICT Group 1 | Daan, Mark & Mike

What This Does

The ESP32 runs in promiscuous WiFi mode, capturing probe requests sent by phones and laptops nearby. Every device that has WiFi on broadcasts these requests to find known networks. The ESP32 logs them and publishes to an MQTT broker, which the live dashboard reads in real time.

What you seeWhat it means
MAC addressUnique hardware ID of the device (often randomised on modern phones)
SSIDNetwork name the device is looking for (wildcard probe = any network)
RSSISignal strength in dBm — stronger = closer
Est. DistanceRough distance estimate using path-loss formula (indoor, ±50% accuracy)

What You Need

Hardware

  • ESP32-WROOM-32 development board
  • USB data cable (not a charge-only cable)
  • A phone or laptop to share a hotspot

Software (one-time install)

  • VS Code — code.visualstudio.com
  • PlatformIO extension — install from VS Code Extensions (search PlatformIO IDE)

✅ No MSYS2, no Python, no manual drivers needed. PlatformIO handles everything automatically.

First-Time Setup

Step 1 — Create the hotspot

The ESP32 is hardcoded to connect to this hotspot:

SettingValue
Network name (SSID)espsniffie
Passwordwifisniffer
Band2.4 GHz (not 5 GHz)

On Android: Settings → Network → Hotspot. On iPhone: Settings → Personal Hotspot. Change the name and password to match exactly.

Step 2 — Open the project in VS Code

Open VS Code → File → Open Folder → select the esp32-sniffer folder (inside this ZIP).

⚠️ The project path must not contain spaces. Place the folder at C:\PlatformIO\esp32-sniffer\ or similar.

Step 3 — First build (downloads toolchain)

Press Ctrl+Alt+B or click the ✓ checkmark in the bottom toolbar.

The first build takes 3–10 minutes — PlatformIO downloads the ESP32 compiler and ESP-IDF automatically. Subsequent builds take about 10 seconds.

✅ You will see [SUCCESS] at the end of the output when it's done.

Step 4 — Flash to ESP32
  1. Plug the ESP32 into USB
  2. Click the → arrow in the bottom PlatformIO toolbar (or Ctrl+Alt+U)
  3. Wait for [SUCCESS] — the ESP32 resets automatically

⚠️ If flashing gets stuck on Connecting.... — hold the BOOT button on the ESP32 until you see Uploading stub..., then release.

Step 5 — Open the dashboard

Open sniffer_dashboard.html in any browser. It connects to the MQTT broker automatically and starts showing devices within a few seconds.

Daily Use

Once flashed, you only need to:

  1. Turn on the espsniffie hotspot on your phone
  2. Plug in the ESP32
  3. Open sniffer_dashboard.html in a browser

No reflashing needed unless you change the code.

Project File Structure

esp32-sniffer/
├── platformio.ini          ← build settings (COM port, board, flash size)
├── sdkconfig.defaults      ← firmware config (WiFi credentials, MQTT, intervals)
├── partitions_spiffs.csv   ← flash partition layout
├── CMakeLists.txt          ← auto-generated by PlatformIO
├── src/
│   ├── main.c              ← main firmware source code
│   ├── CMakeLists.txt      ← declares IDF component dependencies
│   └── Kconfig.projbuild   ← CONFIG_ variable definitions
└── components/
    ├── md5/                ← MD5 hash library (for packet deduplication)
    └── esp-mqtt/           ← MQTT client library (from Espressif GitHub)

Changing Settings

Edit esp32-sniffer/sdkconfig.defaults to change:

SettingKeyDefault
WiFi network nameCONFIG_WIFI_SSIDespsniffie
WiFi passwordCONFIG_WIFI_PSWwifisniffer
ESP32 ID (MQTT topic)CONFIG_ESP32_IDESPSniffie
Room numberCONFIG_ROOM1
Publish interval (seconds)CONFIG_SNIFFING_TIME10

After editing, delete sdkconfig.esp32dev if it exists, then rebuild and reflash.

Troubleshooting

ProblemFix
Flash stuck on Connecting....Hold BOOT button on ESP32 until upload starts
Wrong COM portEdit upload_port = COM5 in platformio.ini to match your port (check Device Manager)
Build error: whitespace in pathMove the project to C:\PlatformIO\esp32-sniffer\ — no spaces allowed in path
Dashboard shows "Connecting..."Open in a browser directly (not via Live Server). Must use wss:// broker.
ESP32 not connecting to hotspotCheck SSID is exactly espsniffie, password wifisniffer, and hotspot is 2.4 GHz
No devices showing on dashboardWait 10–15 seconds after ESP32 boots — first publish takes one interval
